Нужно реализовать небольшой web-app криптокошелька (self-custodial) для TON testnet без собственного backend.
Часть требований намеренно не детализирована. Ожидается, что вы самостоятельно примете решения и опишете их.

Что нужно сделать

Сделайте приложение, в котором пользователь может:

создать новый testnet-кошелек или импортировать существующий;
увидеть адрес кошелька;
увидеть баланс в TON;
увидеть историю транзакций;
получить TON: экран Receive с адресом и копированием адреса;
отправить TON на другой адрес;
получить предупреждения перед отправкой в потенциально рискованных сценариях.

Обязательный функционал
1) Создание / импорт кошелька
создание нового testnet-кошелька;
импорт существующего testnet-кошелька.

2) Главный экран кошелька
После создания или импорта кошелька пользователь должен видеть:
адрес кошелька;
текущий баланс;
список последних транзакций;
поиск по истории транзакций;

3) Receive
На экране получения средств должны быть:
адрес кошелька;
кнопка копирования адреса.

QR-код можно добавить как бонус, но это не обязательно.

4) Send
На экране отправки должны быть:
поле адреса получателя;
поле суммы;
кнопка отправки;
базовая валидация формы.
   
Учтите, что сеть может отвечать не мгновенно. После отправки пользователь должен получить понятный результат.

Дополнительная обязательная фича: Реализовать защиту от адрес-подмены с учётом реалистичных пользовательских сценариев.

Ограничения
Приложение должно быть без собственного backend.
Используйте TON testnet, а не mainnet.
Хранение данных, нужных для UX, можно реализовать локально в браузере.
Так как это тестовое задание, не требуется production-grade реализация безопасности.
UI должен быть минималистичным, но предупреждения должны быть заметны и не игнорироваться пользователем.

В каком виде присылать решение:

1) ссылка на репозиторий;
2) README с инструкцией по запуску;
3) Описание архитектуры, выбранные компромиссы, логика проверок и дальнейшие улучшения приложения.
